Техническая документация программного комплекса
«Система извлечения правил отказов на основе совместной работы мультиагентных систем»
1. Назначение системы

Программный комплекс предназначен для автоматизированного мониторинга состояния промышленного оборудования, раннего обнаружения скрытых дефектов и постановки диагноза (выявления причины отказа).

Система решает проблему диагностики в условиях сложных, многомерных данных, где сбой одного узла вызывает каскадные изменения показаний на множестве различных датчиков. Для решения этой задачи применяется ансамбль из глубокого обучения (нейронные сети), интеллектуального анализа данных (Data Mining) и распределенного искусственного интеллекта (Мультиагентные системы).

2. Общая архитектура

Архитектура системы построена по модульному (микросервисному) принципу и состоит из 5 ключевых подсистем, выполняющихся последовательно:

Модуль сбора и подготовки данных (Генерация и очистка временных рядов).

Модуль предиктивной аналитики (Прогнозирование нормального поведения оборудования).

Модуль обнаружения аномалий (Фиксация отклонений от нормы).

Модуль извлечения знаний (Автоматизированный поиск правил отказов).

Мультиагентная система (Коллаборативный анализ и постановка диагноза в реальном времени).

3. Детальное описание модулей (Подсистем)
3.1. Модуль сбора и препроцессинга (src/data_acquisition)

Отвечает за формирование исходного набора данных.

Симуляция физических процессов: Скрипт data_collector.py генерирует показания 4 датчиков (температура, давление, вибрация, ток мотора). Для обучения системы в данные искусственно внедряется паттерн износа подшипника: резкий скачок вибрации, за которым с временной задержкой следует рост температуры и повышение потребляемого тока.

Очистка данных: Скрипт preprocessor.py применяет метод линейной интерполяции для заполнения пропущенных значений (NaN).

Нормализация: Используется MinMaxScaler из библиотеки scikit-learn для приведения всех показаний к единому масштабу [0, 1], что критически важно для сходимости градиентного спуска при обучении нейросети. Объект масштабирования сохраняется в формате .pkl для применения к будущим (тестовым) данным.

3.2. Модуль прогнозирования временных рядов (src/time_series_prediction)
Ядро машинного обучения системы.
Технология: Рекуррентная нейронная сеть архитектуры LSTM (Long Short-Term Memory).
Принцип работы: Модель обучается исключительно на нормальных данных (до момента возникновения симулированной поломки). Используется метод "скользящего окна" (Sliding Window): сеть получает на вход последние N секунд работы всех датчиков и пытается предсказать их значения на шаг t + 1.
Архитектура сети: 2 слоя LSTM (с механизмом Dropout для предотвращения переобучения) и выходной полносвязный слой (Dense). Функция потерь — MSE (среднеквадратичная ошибка).


3.3. Модуль обнаружения отказов (src/fault_detection)
Связующее звено между математикой нейросетей и логикой агентов.
Формирование ошибки: Через обученную LSTM-сеть пропускается весь набор данных. Вычисляется абсолютная разница между фактическим значением датчика и тем, что предсказала сеть (Reconstruction Error).
Динамические пороги (threshold_manager.py): Система автоматически вычисляет среднюю ошибку стандартное отклонение на эталонном (нормальном) участке данных. Порог аномалии устанавливается как Threshold=μ+3σ (правило трех сигм).
Выход: Если ошибка предсказания превышает порог, временная точка помечается бинарным флагом 1 (Аномалия). Формируется датафрейм событий.

3.4. Модуль извлечения правил отказов (src/fault_rule_extraction)

Реализует концепцию Data Mining для формирования базы знаний.

Технология: Алгоритм поиска ассоциативных правил Apriori (библиотека mlxtend).

Логика: Алгоритм разбивает временную шкалу аномалий на "окна" (например, по 5 минут). Если внутри одного окна срабатывают несколько датчиков, алгоритм фиксирует совместную встречаемость (itemset).

Метрики: Извлекаются правила формата IF (Условие) -> THEN (Следствие). Правила фильтруются по метрике Confidence (Достоверность) — вероятности того, что при поломке компонента А обязательно выйдет из строя компонент Б.

Результат: Формируется файл базы знаний extracted_fault_rules.csv, понятный как машине, так и человеку.

3.5. Мультиагентная система (src/multi_agent_system)

Распределенная система для анализа потоковых данных.

Паттерн взаимодействия: Event-Driven Architecture (Событийно-ориентированная архитектура) через внутреннюю шину сообщений EventBus (Издатель-Подписчик).

Типы агентов:

DataMonitoringAgent (Агенты мониторинга): Привязаны к конкретным датчикам. Следят за потоком флагов аномалий. Публикуют событие в шину только в момент изменения состояния (норма -> сбой, или сбой -> норма), снижая нагрузку на сеть.

CorrelationAgent (Агент-Анализатор): Подписан на всех агентов мониторинга. Аккумулирует события во времени. Если замечает, что сбоят N>1 датчиков в пределах заданного временного окна, публикует событие "Обнаружена Корреляция".

CoordinationDecisionAgent (Главный агент/CEO): Подписан на Агента-Анализатора. Имеет доступ к Базе Знаний (результатам работы алгоритма Apriori). При получении корреляции, сопоставляет список сбойных датчиков с условиями в Базе Знаний и выдает финальный диагноз с указанием процента достоверности и рекомендациями для ремонтной бригады.

4. Конфигурация системы

Глобальные параметры системы вынесены в файл src/config.py. Это позволяет перенастраивать систему без вмешательства в исходный код.
Основные настраиваемые параметры:

SEQUENCE_LENGTH — размер окна памяти для LSTM (глубина погружения в историю).

ANOMALY_THRESHOLD_MULTIPLIER — чувствительность детектора аномалий (множитель сигмы).

MIN_SUPPORT и MIN_CONFIDENCE — жесткость алгоритма Apriori при фильтрации извлекаемых правил.

5. Инструкция по запуску (Deployment)

Требования:

Python 3.9 - 3.12

Установленные библиотеки (см. requirements.txt)

Шаги запуска:

Развернуть виртуальное окружение и установить зависимости:

pip install numpy pandas scikit-learn tensorflow mlxtend

Для запуска полного конвейера (сбор данных -> обучение -> поиск правил -> запуск МАС) выполните команду из корневой директории:

python main.py

(Примечание: процесс обучения LSTM-сети может занять несколько минут в зависимости от вычислительной мощности устройства. Результаты работы агентов и поставленные диагнозы будут выводиться непосредственно в консоль).
